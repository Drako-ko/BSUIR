23. Конфликт - противоречия в разных ветках, например в одном и том же месте в файле разные слова,т.е коммиты в двух разных ветках противоречат друг другу и мы мержим(сливаем) ветки.
В случае несовпадения содержимого одинаковых вайлов из разных веток. Просто поменять содержимое файла(23)
git log --all --graph(для вывода с графиком пересечений(можно hist --all))
Для устранения конфликта после merge делаем новый коммит предварительно отменив изменения
git merge --abort 
33. Забрать изменения удаленной ветки из репозитория по умолчанию,  основной ветки   
git fetch [имя удал. сервера] (не сливает с вашими наработками, забирает данные, которых у вас нет, (чтобы слить git merge), т.е. добавление с промежуточным репозиторием)
git pull (автоматически извлекает и сливает данные из удалённой ветки в текущую ветку, что-то типа fetch+merge)
